Q1 (DRY — Part 3).
    Quote the exact lines or expressions that were duplicated between your
    Part 2 copy constructor and copy assignment. Which of the two refactors
    did you choose (helper method, or copy-and-swap), and how does it
    eliminate that duplication? If you chose copy-and-swap, also explain
    why it is exception-safe.

    if(this==&other){
        return;
    }

    delete[] _friends;
    _capacity = other.capacity;
    _size = other.size;
    _friends = new std::string[_capacity];
    for(int i =0;i<_size;i++){
        _friends[i] = other._friends[i];
    }

    I used copy and swap idiom. Copy and swap idiom implicity uses the copy constructor by taking in the parameter as value instead of reference. This means two things for the program. Firstly, no need to write any new statements or initiate objects on the heap. Instead, we can simply swap values beteen the current object and the argument. The second reason is that this is exception safe. The reason is, the allocation of memory for the array of strings happens before the current data is deleted, thus, if there is a bad allocation, the original data is retained. In fact, we do not even need to delete any data manually as the destructor of newly created object from the argument takes care of that.
